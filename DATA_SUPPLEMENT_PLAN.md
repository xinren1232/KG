### 2.3 需要补充的关系类型

#### 新增关系类型（5种）
```
1. CAUSED_BY (被...引起)
   Symptom --[CAUSED_BY]--> RootCause
   示例: 黑屏 --[CAUSED_BY]--> 屏幕排线断裂

2. RESOLVED_BY (被...解决)
   Symptom --[RESOLVED_BY]--> Solution
   示例: 对焦失败 --[RESOLVED_BY]--> 更换VCM马达

3. PREVENTS (预防)
   Solution --[PREVENTS]--> Symptom
   示例: 定期清洁镜头 --[PREVENTS]--> 拍照模糊

4. DETECTS (检测)
   TestCase --[DETECTS]--> Symptom
   示例: AF成功率测试 --[DETECTS]--> 对焦失败

5. HAS_FAILURE_MODE (有失效模式)
   Component --[HAS_FAILURE_MODE]--> RootCause
   示例: FPC --[HAS_FAILURE_MODE]--> 弯折断裂
```

#### 需要加强的现有关系
```
1. AFFECTS (影响) - 当前较少
   目标: 每个Component至少关联3个Symptom
   示例: 摄像头 --[AFFECTS]--> 对焦失败/拍照模糊/摄像头黑屏

2. USES (使用) - 当前较少
   目标: 每个TestCase至少关联2个Tool
   示例: 分辨率测试 --[USES]--> 测试图卡/光学测试设备

3. BELONGS_TO (归属) - 需要完善
   目标: 所有术语都有明确归属
   示例: VCM马达 --[BELONGS_TO]--> 摄像头模组
```
