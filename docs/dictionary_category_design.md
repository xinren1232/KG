# 词典类别设计文档

## 1. 类别体系架构

### 1.1 三级分类体系
```
Level 1: 主类别 (3个)
├── 硬件组件 (Components)
├── 异常现象 (Symptoms)  
└── 工具方法 (Tools)

Level 2: 子类别 (按功能域划分)
Level 3: 具体标签 (多维度标记)
```

### 1.2 主类别定义

#### 硬件组件 (Components)
- **定义**: 手机产品中的所有物理部件、模组和材料
- **范围**: 从芯片级器件到整机结构件
- **用途**: 支持产品设计、供应链管理、质量控制

#### 异常现象 (Symptoms)
- **定义**: 产品在设计、制造、使用过程中出现的各种异常表现
- **范围**: 外观缺陷、功能异常、可靠性问题
- **用途**: 支持问题诊断、质量分析、改进决策

#### 工具方法 (Tools)
- **定义**: 用于测试、分析、改进的各种工具、设备和方法
- **范围**: 测试设备、分析方法、改进措施、标准规范
- **用途**: 支持质量保证、问题解决、工艺优化

## 2. 子类别划分

### 2.1 硬件组件子类别
- **电子器件**: PCB、SMT、芯片、连接器、传感器
- **机械结构**: 外壳、按键、转轴、密封、紧固件
- **功能模组**: 摄像头、显示、电池、天线、马达
- **材料工艺**: CMF、涂层、胶粘剂、填充材料

### 2.2 异常现象子类别
- **外观缺陷**: 划伤、色差、间隙、变形、污染
- **功能异常**: 失效、性能下降、兼容性问题
- **可靠性问题**: 老化、疲劳、腐蚀、磨损

### 2.3 工具方法子类别
- **测试设备**: 各种检测、测量、分析仪器
- **分析方法**: 失效分析、统计分析、仿真分析
- **改进措施**: 工艺优化、设计改进、质量控制

## 3. 标签体系

### 3.1 功能域标签
- 硬件相关、软件相关、结构相关
- 显示相关、影像相关、通信相关
- 电池相关、声学相关、热管理

### 3.2 工艺标签
- 制造工艺、测试验证、设计
- SMT、注塑、装配、点胶
- EMC、可靠性、安全相关

### 3.3 应用标签
- 质量体系、供应链、用户体验
- 成本控制、效率提升、标准化

## 4. 使用指南

### 4.1 分类原则
1. **唯一性**: 每个术语归属唯一主类别
2. **完整性**: 覆盖手机制造全生命周期
3. **可扩展**: 支持新技术和新工艺
4. **实用性**: 便于检索和关联分析

### 4.2 标签使用
1. **多维标记**: 一个术语可有多个标签
2. **层次结构**: 从通用到具体的标签层次
3. **关联性**: 支持跨类别的关联查询
4. **标准化**: 使用统一的标签词汇表

## 5. 维护机制

### 5.1 版本管理
- 类别变更需要版本记录
- 向后兼容性保证
- 变更影响评估

### 5.2 质量控制
- 新增术语的分类审核
- 定期的分类一致性检查
- 用户反馈的分类优化
