# 词典图谱设计规范

## 📋 总体架构

### 🎯 设计目标
- **标准化词典**: 建立统一的术语标准，支持多维度分类和标签
- **知识图谱**: 构建实体关系网络，支持复杂查询和推理
- **可扩展性**: 支持后续领域扩展（摄像、显示、射频等）

### 🏗️ 核心架构
```
词典系统 (Dictionary) → 图谱节点 (Graph Nodes) → 关系网络 (Relationships)
     ↓                        ↓                         ↓
  标准术语库              实体+属性+标签              业务关联关系
```

## 🏷️ 标签体系设计

### 📊 标签分组 (70个标签，5个组)
| 组别 | 标签数量 | 用途 | 示例标签 |
|------|----------|------|----------|
| **domain** | 13个 | 功能域 | 显示相关、影像相关、声学、射频相关 |
| **process** | 13个 | 工艺流程 | 制造工艺、SMT、注塑、测试验证 |
| **quality** | 6个 | 质量相关 | 可靠性、ESD、质量体系、外观 |
| **lifecycle** | 6个 | 生命周期 | 设计、硬件相关、软件相关、项目相关 |
| **material** | 6个 | 物料相关 | 物料、CMF、半导体、电气性能 |
| **function** | 4个 | 功能相关 | 功能、人机交互、用户体验、配件 |
| **organization** | 3个 | 组织相关 | 组织职责、供应链、工具 |
| **other** | 19个 | 其他 | ICT、线缆管理、系统稳定性等 |

### 🎯 标签使用原则
- **多维度**: 每个术语可以有多个标签
- **横切关注**: 标签跨越Label边界
- **语义明确**: 每个标签有明确的业务含义
- **可扩展**: 支持新增标签和标签组

## 📝 数据录入模板

### 🔤 CSV格式模板
```csv
term,aliases,category,sub_category,tags,definition,example,source,status,updated_at
示波器,Oscilloscope;OSC,Tool,检测,工具;测试验证;电气性能,用于观察电信号波形的电子测试仪器,测试电源纹波时使用,标准化词典,active,2025-09-26
```

### 📋 字段填写规范
- **term**: 使用标准中文术语，避免缩写
- **aliases**: 用分号分隔，包含英文名、缩写、同义词
- **category**: 必须是8个Label之一
- **sub_category**: 参考映射表，保持一致性
- **tags**: 用分号分隔，从70个标签白名单中选择
- **definition**: 简洁准确的标准定义，50-200字
- **example**: 具体使用场景或示例
- **source**: 数据来源标识
- **status**: 通常为"active"
- **updated_at**: 自动生成或手动填写

## 🔗 关系设计 (未来扩展)

### 📊 核心关系类型
```cypher
// 异常相关
(Anomaly)-[:HAS_SYMPTOM]->(Symptom)
(Anomaly)-[:AFFECTS]->(Component)
(Anomaly)-[:RESOLVED_BY]->(Solution)

// 测试相关
(TestCase)-[:TESTS]->(Component)
(TestCase)-[:USES_TOOL]->(Tool)

// 流程相关
(Process)-[:USES_MATERIAL]->(Material)
(Role)-[:OWNS_PROCESS]->(Process)

// 产品相关
(DeviceModel)-[:HAS_COMPONENT]->(Component)
```

## 🎯 质量标准

### ✅ 数据质量要求
- **完整性**: 必填字段100%填写
- **准确性**: 定义准确，无歧义
- **一致性**: 同类术语格式统一
- **标准化**: 遵循行业标准术语

### 🔍 验证规则
- **唯一性**: 同Label下术语名称唯一
- **标签白名单**: 标签必须在预定义列表中
- **分类约束**: category必须是8个Label之一
- **格式规范**: 别名用分号分隔，标签用分号分隔
