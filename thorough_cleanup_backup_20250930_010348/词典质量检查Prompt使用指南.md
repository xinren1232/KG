# 📋 词典质量检查Prompt使用指南

## 🎯 功能概述

**词典质量检查专家**是一个专业的AI助手prompt，用于对手机研发质量词典进行全面的质量检查和评估。它集成了四大检查维度，能够快速识别问题并提供具体的改进建议。

---

## 🔧 使用方法

### 📍 **访问路径**
1. 打开前端系统: `http://localhost:5173`
2. 导航到: `系统管理` → `Prompt管理`
3. 选择分类: `分析评估 (analysis)`
4. 找到: `词典质量检查专家`

### 📝 **使用步骤**

#### 第一步: 准备词典数据
```
支持格式:
├── CSV格式 (推荐)
├── 表格格式 (Markdown)
├── JSON格式
└── 纯文本格式
```

#### 第二步: 填写参数
- **dictionary_content**: 粘贴完整的词典数据
- **check_focus**: 指定检查重点 (可选)
- **quality_standard**: 设定质量标准 (可选)

#### 第三步: 生成检查prompt
点击"测试"或"生成"按钮，系统自动生成完整的检查prompt

#### 第四步: 执行检查
将生成的prompt提供给AI助手，获得详细的质量检查报告

---

## 🔍 检查维度详解

### 1. 📊 **基础规范检查**
#### 检查内容
- **字段完整性**: 术语、别名、类别、多标签、备注五个字段
- **格式规范**: 分隔符使用、字段长度限制
- **分类标准**: 8大标准分类的准确性

#### 常见问题
- 缺少必填字段
- 分类不在标准范围内
- 格式不符合规范

### 2. 🧠 **内容逻辑检查**
#### 检查内容
- **技术准确性**: 术语定义的专业性和准确性
- **分类匹配**: 类别与术语定义的相关性
- **标签相关**: 多标签与术语应用领域的匹配

#### 常见问题
- 分类与术语特征不匹配
- 标签选择不合理
- 技术描述不准确

### 3. 🔄 **重复冲突检查**
#### 检查内容
- **术语重复**: 相同术语名称的重复
- **别名冲突**: 不同术语共享相同别名
- **语义重复**: 含义相同但表达不同的术语

#### 常见问题
- 完全相同的术语重复
- 别名在多个术语间冲突
- 语义重复但表达不同

### 4. 💎 **实用价值检查**
#### 检查内容
- **信息丰富度**: 描述的详细程度和价值
- **应用场景**: 是否包含实际应用信息
- **问题导向**: 对症状类术语的原因分析

#### 常见问题
- 描述过于简单
- 缺少应用场景说明
- 症状类缺少原因分析

## 📊 输出报告解读

### 🚨 **严重问题 (必须修复)**
- **影响**: 影响词典基本可用性
- **优先级**: 最高
- **处理**: 立即修复

### ⚠️ **警告问题 (建议修复)**
- **影响**: 影响词典质量和专业性
- **优先级**: 中等
- **处理**: 尽快修复

### 💡 **信息提示 (优化建议)**
- **影响**: 影响词典的完善程度
- **优先级**: 较低
- **处理**: 持续改进

### 📈 **质量评分标准**
- **9-10分**: 优秀 - 完全符合专业标准
- **7-8分**: 良好 - 基本符合标准，少量问题
- **5-6分**: 一般 - 符合基本要求，需要改进
- **3-4分**: 较差 - 存在明显问题
- **1-2分**: 很差 - 严重不符合标准

## 💡 使用技巧

### 🎯 **检查重点设置**
```
常用检查重点:
├── "重点检查分类准确性" - 专注分类问题
├── "重点检查重复冲突" - 专注去重
├── "重点检查实用价值" - 专注内容质量
├── "全面质量检查" - 四维度全面检查
└── "新增数据检查" - 针对新导入数据
```

### 📋 **质量标准设置**
```
质量标准选项:
├── "企业级标准" - 最高标准
├── "行业标准" - 行业通用标准
├── "基础标准" - 基本可用标准
└── "自定义标准" - 特定要求
```

### 🔧 **批量检查策略**
1. **分批检查**: 大量数据分批次检查
2. **重点检查**: 针对特定问题类型检查
3. **增量检查**: 只检查新增或修改的数据
4. **定期检查**: 建立定期质量检查机制
